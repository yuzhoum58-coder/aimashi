## 1. 产品定位

替代 Mimics 26 的部分功能，专注**骨科手术规划**场景：

1. **多模态配准** — CT + MRI 手动/自动刚体配准
2. **分割建模** — 阈值分割 + 三维网格导出
3. **手术规划** — 模拟置钉 + 测量

**MVP范围**: 只做第1条（配准），第2-3条为框架预留。

## 3. 交互规范

### 3.1 四视图交互行为

| 视口 | 左键拖拽 | 滚轮 | 右键拖拽 | 中键 |
|------|---------|------|---------|------|
| **Axial** | 平移切片 | 切换切片(前后) | 窗宽窗位(水平=窗宽,垂直=窗位) | 重置视图 |
| **Sagittal** | 平移切片 | 切换切片 | 窗宽窗位 | 重置视图 |
| **Coronal** | 平移切片 | 切换切片 | 窗宽窗位 | 重置视图 |
| **3D** | 旋转模型 | 缩放 | 平移 | 重置视图 |

### 3.2 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+O` | 打开DICOM文件夹 |
| `Ctrl+W` | 关闭当前序列 |
| `Ctrl+S` | 保存项目文件 (.aimashi) |
| `Ctrl+E` | 导出变换矩阵 (.tfm / .json) |
| `Ctrl+R` | 重置配准(归零) |
| `Space` | 切换配准面板显示/隐藏 |
| `Tab` | 切换叠加模式 (半透明/棋盘格/差值/关) |
| `↑/↓` | 切片前后滚动 (1步) |
| `PgUp/PgDn` | 切片前后滚动 (10步) |
| `Home/End` | 跳到首/末切片 |
| `1/2/3/4` | 切换到单个视口全屏(1=Axial,2=Sagittal,3=Coronal,4=3D) |
| `Esc` | 退出单视口全屏,回到四视图 |
| `Delete` | 移除浮动序列 |
| `F11` | 全屏模式 |

### 3.3 配准面板交互

**6个滑块的行为：**

| 滑块 | 范围 | 精度 | 默认 | 单位 |
|------|------|------|------|------|
| Tx (X平移) | ±100 | 0.1 | 0 | mm |
| Ty (Y平移) | ±100 | 0.1 | 0 | mm |
| Tz (Z平移) | ±50 | 0.1 | 0 | mm |
| Rx (X旋转) | ±180 | 0.5 | 0 | 度 |
| Ry (Y旋转) | ±180 | 0.5 | 0 | 度 |
| Rz (Z旋转) | ±180 | 0.5 | 0 | 度 |

**按钮行为：**

| 按钮 | 行为 |
|------|------|
| [重置] | 6个滑块归零，MRI恢复原始位置 |
| [确认] | 当前变换矩阵应用于浮动序列，记录为"已配准" |
| [导出] | 弹出保存对话框，导出变换矩阵 |
| 浮动序列下拉 | 切换浮动序列（如果有3个以上序列） |
| 叠加模式下拉 | 半透明/棋盘格/差值图 |

**滑块联动逻辑：**
```
滑块移动 → 生成4×4变换矩阵 (Euler3DTransform)
          → MRI vtkImageReslice.SetResliceAxes(matrix)
          → Axial视口的MRI半透明层按矩阵重采样
          → 三视图同步Render()
```

**实时性要求：** 滑块拖动到最终位置到图像更新应 < 100ms（实测2ms/帧，满足要求）

**叠加模式：**

| 模式 | 显示 | 用途 |
|------|------|------|
| 半透明 | CT灰阶 + MRI伪彩色半透明 | 日常配准操作 |
| 棋盘格 | 棋盘格交替显示CT/MRI | 检查对齐精度 |
| 差值图 | CT-MRI相减(归一化后) | 评估一致性 |
| 仅CT | 只显示CT | 参考 |
| 关 | 关闭叠加 | 查看原始CT |

## 4. 操作流程

### 4.1 主流程

```
Step 1: 打开DICOM
  文件 → 打开 (Ctrl+O)
  → 弹出文件夹选择对话框
  → 扫描文件夹内所有DICOM序列
  → 按 SeriesInstanceUID 分组列出
  → 自动识别 Modality (CT/MR)

Step 2: 选择序列
  → 用户选择"主序列"(例如CT)
  → 用户选择"浮动序列"(例如MRI)
  → 两个序列同时加载到四视图

Step 3: 初始对齐（可选）
  → 系统自动计算初始对齐（基于图像质心）
  → 或从上次保存的.tfm文件导入

Step 4: 手动配准
  → 用户拖动6个滑块
  → MRI半透明层实时跟随变换
  → 切换到棋盘格验证对齐
  → 用户确认配准

Step 5: 导出/保存
  → 导出变换矩阵 (.tfm / .json / 文本)
  → 保存项目文件 (.aimashi 格式)
  → 继续下一步（分割/建模）
```

### 4.2 文件操作

**打开流程细节：**

```
用户点击"打开" (Ctrl+O)
  → 弹出 QFileDialog.getExistingDirectory()
  → DicomLoader.scan_folder(path)
    → 递归扫描所有.dcm文件
    → pydicom读取每个文件的SeriesInstanceUID
    → 按UID分组
    → 每组检查Modality/PatientName/SeriesDescription
  → 弹出序列选择对话框:
    ┌──────────────────────────────────────┐
    │ 找到 N 个DICOM序列:                   │
    │                                      │
    │ □ [1] CT Abdomen (64 slices, CT)     │
    │ ☑ [2] MRI T1 (64 slices, MR)         │
    │ □ [3] MRI T2 (32 slices, MR)         │
    │                                      │
    │ 主序列: [1]   浮动序列: [2]           │
    │                        [加载] [取消]  │
    └──────────────────────────────────────┘
```

**变换矩阵导出格式 (.tfm)：**

```json
{
  "version": "1.0",
  "created": "2026-05-29T12:00:00",
  "fixed_series": {
    "uid": "1.2.840.xxx",
    "modality": "CT",
    "description": "CT Abdomen",
    "patient": "Test^CT"
  },
  "moving_series": {
    "uid": "1.2.840.xxx",
    "modality": "MR",
    "description": "MRI T1",
    "patient": "Test^MRI"
  },
  "transform_type": "Euler3D",
  "translation_mm": [5.0, 3.0, 2.0],
  "rotation_deg": [0.5, 0.3, 0.1],
  "matrix_4x4": [
    [0.999, 0.000, 0.008, 5.000],
    [0.000, 0.999, 0.005, 3.000],
    [-0.008, -0.005, 0.999, 2.000],
    [0.000, 0.000, 0.000, 1.000]
  ]
}
```

### 4.3 文件格式支持

| 格式 | 读 | 写 | 说明 |
|------|----|----|------|
| DICOM (.dcm) | ✅ | ❌ | 作为输入，不修改原始文件 |
| NIfTI (.nii/.nii.gz) | ⏳ Phase 2 | ❌ | SimpleITK原生支持 |
| NRRD (.nrrd) | ⏳ Phase 2 | ❌ | SimpleITK原生支持 |

## 5. 数据流

### 5.1 DICOM → 渲染管线

```
DICOM文件(.dcm)
  │
  ▼
SimpleITK.ImageSeriesReader
  ├─ GetGDCMSeriesFileNames() → 文件列表(按InstanceNumber排序)
  └─ Execute() → sitk.Image (物理坐标空间)
  │
  ▼
sitk.GetArrayFromImage() → numpy.ndarray (Z,Y,X)
  │
  ▼
vtk.vtkImageData (通过 numpy_support.numpy_to_vtk)
  ├─ SetDimensions(X,Y,Z)     ← transpose(Z,Y,X) → (X,Y,Z)
  ├─ SetSpacing(x,y,z)
  └─ SetOrigin(origin_x, origin_y, origin_z)
  │
  ▼
vtkImageReslice (切片引擎)
  ├─ SetInputData(volume)
  ├─ SetResliceAxes(transform_matrix)  ← 手动配准时更新
  └─ Update() → 2D切片
  │
  ▼
vtkImageMapper / vtkImageActor
  │
  ▼
vtkRenderer (视口)
  └─ vtkRenderWindow → 显示
```

### 5.2 手动配准数据流

```
滑块事件 (值变化)
  │
  ▼
Euler3DTransform  ← Tx,Ty,Tz,Rx,Ry,Rz
  │
  ├─ 生成 4×4 变换矩阵 (vtkMatrix4x4)
  │
  ▼
vtkImageReslice (MRI volume)
  ├─ SetResliceAxes(matrix)
  └─ Update()
  │
  ▼
MRI切片 (变频矩阵后的浮动图像)
  │
  └─ Axial视口: CT底图 + MRI半透明 → 同时Render()
  └─ Sagittal视口: CT底图 + MRI半透明（可选）
  └─ Coronal视口: CT底图 + MRI半透明（可选）
  │
  ▼
用户判断对齐效果
  ├─ 继续调整 → 回到滑块事件
  └─ 确认 → 记录当前矩阵
```

### 5.3 叠加模式数据流

```
半透明模式:
  CT Actor (opacity=1.0) + MRI Actor (opacity=0.4)

棋盘格模式:
  CT/MRI 各取一半像素 → 交替排列成棋盘格
  → 生成新的vtkImageData → 替换Actor
  
差值图模式:
  CT - MRI(归一化后) → 差异图像
  → 生成新的vtkImageData → 替换Actor
```

### 6.2 配准面板状态

| 状态 | 显示 | 意义 |
|------|------|------|
| 未加载 | 面板灰色，所有控件禁用 | 无DICOM加载 |

| 已配准 | 矩阵锁定(确认后)，底色变绿 | 配准完成 |
| 已导出 | 状态栏提示"矩阵已导出" | 已保存到文件 |
